Real-Time Multilevel Terrain-Aware Path Planning for Ground Mobile Robots in Large-Scale Rough Terrains

Real-Time Multilevel Terrain-Aware Path Planning for Ground Mobile Robots in Large-Scale Rough Terrains

问题非常工程化：带履带与辅助鳍（Flippers）的机器人在野外坑洼、废墟乃至多层建筑中，如何规划路径才能既快又不翻车。下文按动机、算法与实验拆解。

一、Motivation

痛点 1：地图维度的局限

许多方法依赖 2.5D 高程图（Elevation Map）（类似在地形上罩一层高度场）。这类表示难以正确处理桥洞、多层楼梯等真正的 3D 多层结构。

痛点 2：过度简化导致失稳

为追求速度，常有做法把履带车骨架化成少数线段/简化模型。在碎石废墟等场景里，接触点估计稍有偏差，就可能导致重心判断错误进而翻车。

二、核心算法与关键公式（Methodology）

整体为双层架构：全局层（在大地图里算可通行性与路径）+ 局部层（调整履带/鳍片姿态，防倾覆）。

1. 全局层：隐式地图上的地形分析

为在大范围兼顾效率与几何信息，作者在较粗分辨率的体素中引入基于 NDT（Normal Distribution Transform）的表示：每个体素内用 3D 高斯分布拟合局部几何。可与深度学习类比：在低分辨率栅格里仍保留分布级的高维形状信息，而非仅一个标量高度。

对协方差矩阵做特征分解，设特征值为 \(\lambda_1 \le \lambda_2 \le \lambda_3\)，对应特征向量刻画局部主方向；\(\lambda_1\) 与最小尺度上的“厚薄/平坦程度”相关。

粗糙度（Roughness）

$$ r(v) = 1 - \frac{\lambda_2 - \lambda_1}{\lambda_2 + \lambda_1} $$

若 \(\lambda_1 \ll \lambda_2\)，局部更接近平面，\(r(v)\) 较小（更平坦）；若 \(\lambda_1\) 与 \(\lambda_2\) 接近，起伏更剧烈，粗糙度升高。

坡度（Slope）

令 \(\mathbf{n}_1\) 为与 \(\lambda_1\) 对应的特征方向所给出的地面法向（论文中的约定），\(\mathbf{n}_z\) 为与重力相反的单位向量。则

$$ s(v) = \arccos\left(\frac{\mathbf{n}_1 \cdot \mathbf{n}_z}{\lVert\mathbf{n}_1\rVert\,\lVert\mathbf{n}_z\rVert}\right) $$

夹角越大，局部坡面越陡。

地形复杂度（Terrain Complexity）

将粗糙度、坡度与稀疏度 \(d(v)\)（论文中对点云/占据稀疏程度的度量）归一化后加权：

$$ \tau(v) = \alpha_r \frac{r(v)}{r_{\mathrm{crit}}} + \alpha_s \frac{s(v)}{s_{\mathrm{crit}}} + \alpha_d \frac{d(v)}{d_{\mathrm{crit}}} $$

再与阈值比较，判断体素 \(v\) 是否可安全通行。

2. 局部层：位形–稳定性迭代评估

全局给出朝向后，局部需确定履带与鳍片等位形，使机体不翻。

  • ZMP / 支撑多边形思路：将所有地面接触点凸包成支撑多边形（Support Polygon）；将质心（CoM）沿重力方向投影到地面。若投影落在多边形内部，则在该简化判据下视为稳定。
  • 迭代修正：若投影落在外侧，则寻找多边形边界上的倾覆轴（Tip-over axis），在模型中绕该轴做小幅“虚拟转动”，更新接触与多边形，直至稳定或判定不可行——用几何迭代替代高成本完整物理仿真。

三、Experiments

实验 1：地形指标精度

  • 真值：以约 0.025 m 分辨率的精细点云 / Surfel 地图上算得的粗糙度为参考。
  • 对比：传统 Grid / 纯体素在 0.1 m、0.2 m 等粗分辨率下，粗糙度与真值偏差大（细节被栅格抹平）。
  • 本文（如 Ours-0.2m）：借助 NDT 隐式分布，即使在 0.2 m 粗体素上，粗糙度与高精度图的相关性仍接近 1.0。

实验 2：全局路径（3D 多层）

  • 场景：多层楼梯、大型桥洞等。
  • ECTN 等 2.5D 方法：桥上下被压成一层，桥下易被当成不可通，路径绕远。
  • 本文：保留 3D 多层结构，可走桥洞等捷径，路径更短。

实验 3：实车稳定性（60 条高难度路线）

  • 场景：碎石、废墟、凸起、楼梯等。
  • 骨架化 CS 方法：通过率约 60.0%（接触估计不准易卡死）。
  • 极简动态规划 GEO：约 68.0%。
  • 本文：约 90.0%；假阳性（判安全却翻车）与假阴性（实际可走却判不可走）各仅约 3 次（以论文报告为准）。

四、Takeaway

全局：用高斯/NDT在粗体素里保留几何统计细节，避免只靠标量高程。局部：用支撑多边形 + 质心投影 + 倾覆轴迭代做可实时运行的稳定性估计，而非依赖重型物理引擎黑箱仿真。整体是线性代数与优化（特征分解、法向、凸包与几何迭代）驱动的工程化路径规划，而非数据驱动深度学习。
